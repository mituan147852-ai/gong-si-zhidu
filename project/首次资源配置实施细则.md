## 核心条款（要点充沛，引用原文关键表述）

### 第X条（目的）

> "贯彻公司'一创三领先'发展战略，切实发挥'项目实施策划'对项目履约的创效作用，提高项目首次资源配置的时效性和针对性，为项目履约提供强有力的资源保障，防范因资源配置不合理导致的重大风险。"

### 第X条（管理原则）

| 原则 | 说明 |
|------|------|
| 法人管项目原则 | 公司作为责任主体，充分发挥企业资源平台的作用，加强项目实施策划的前端管理，主导项目的首次资源配置工作 |
| 核心配置原则 | 首次资源配置应围绕核心管理资源配置、核心技术资源配置、核心生产资源配置工作开展，防范工程项目发生重大风险 |
| 合理配置原则 | 首次资源配置应根据项目合同要求及项目的具体特点，结合以往施工经验，在充分对比的基础上，对核心资源进行合理配置 |
| 全周期保障原则 | 首次资源配置将立足于项目履约全周期，保障效果应能够贯穿项目启动/实施/收尾/维保阶段，禁止在项目启动阶段进行阶段性和临时性配置 |
| 责任追究原则 | 因首次资源配置工作失误造成履约过程中重大经济损失和品牌遭受影响的情况，将严格追究责任 |
| 分级管理原则 | 总部牵头重大项目/重点管控项目的首次资源配置工作，分公司/事业部负责一般项目的首次资源配置工作 |

### 第X条（术语定义）

| 术语 | 定义 |
|------|------|
| 首次资源 | 在项目正式施工前，子企业根据项目特点，为保证项目全周期正常履约所配置的有针对性的首要资源，分为核心管理资源、核心技术资源、核心生产资源三类 |
| 核心管理资源 | 项目经理和项目班子成员 |
| 核心技术资源 | 施工部署、主要施工方案、科技成果、技术范例等 |
| 核心生产资源 | 主要物资、大型施工设备、项目资金以及主要劳务分包和专业分包 |

## 四维宣贯表

### 做什么（要点充沛，8-15条）

1. **联络机制建立**：公司总部/分公司/事业部/项目部均须设置首次资源配置联络员一名，负责组织协调和资料传递。
2. **项目分类管理**：根据项目特点、难点、重要程度、风险程度分为重大项目（国家保障性/局重点/公司重点）、一般项目（常规工程）、重点管控项目（风险较高工程）。
3. **核心管理资源配置**：确定项目经理和项目班子成员任职条件；从人才库筛选（具备同类工程经验且业绩较好优先选聘）；输出核心管理人员汇总表。
4. **核心技术资源配置**：以公司现有科技成果资源平台为依托提取/借鉴相关技术成果；编制主要施工方案清单；明确关键要素；提供核心生产资源配置的技术条件；调查项目属地条件（行政管理要求/地质水位条件等）。
5. **核心生产资源配置——分包资源**：优先选择现有资源平台中的合格分供/分包商；特殊情况需新引进时须严格按资源引入程序考察/审核/准入后方可参与采购招投标；重点统筹劳务分包使用避免一家优质分包被多个项目同时使用。
6. **核心生产资源配置——物资设备**：建立生产资源采购平台维护和统筹管理；分析采购数据及市场变化趋势；指导/监督集中采购/区域联采工作。
7. **属地资源调查**：项目团队在首次资源配置前应调查项目属地资源情况（行政管理要求/各类生产资源和分包资源的渠道/价格/同类项目资源配置习惯等）。
8. **履约风险识别**：项目团队在首次资源配置前认真开展项目履约风险及重点/难点识别工作。
9. **检查及评价**：项目部随时关注配置资源的使用状态；分公司/事业部每季度对所辖项目首次资源配置应用效果进行检查评价；公司总部每半年开展一次检查评价；每年对当年竣工项目资源配置应用效果进行汇总考核。
10. **倒查机制**：如项目部发生重大投诉或其他影响企业品牌的履约管理问题，公司总部将适时对其首次资源配置管理开展倒查工作。
11. **资源库建设与维护**：公司总部项目管理部负责核心管理资源库建立和维护；各分公司/事业部搭建分包资源/分供资源子平台，与公司资源平台对接。
12. **项目核心生产资源决策**：项目核心生产资源配置应由项目团队核心成员共同决策，并留有决策记录。
13. **首次资源配置汇总表评审**：分公司/事业部牵头项目团队完成《首次资源配置汇总表》的评审。
14. **资源配置效果分析**：项目部在每个子分部精益建造措施实施完毕后15天内完成实施效果分析并编制分析报告。
15. **汇总考核**：公司总部每年对当年竣工项目资源配置应用效果进行汇总考核。

### 怎么做（操作流程详细展开，10-20步）

#### 一、项目分类与联络机制

**步骤1：项目分类确定**
- 发起部门：公司总部
- 操作动作：根据项目特点/难点/重要程度/风险程度确定项目分类
  - 重大项目：国家保障性工程/局重点工程/公司重点工程
  - 一般项目：常规工程
  - 重点管控项目：风险较高工程
- 输出文件：项目分类清单
- 签字审批：公司总部

**步骤2：联络员设置**
- 发起部门：公司总部/分公司/事业部/项目部
- 操作动作：均须设置首次资源配置联络员一名
- 职责：负责首次资源配置管理过程中的组织协调和资料传递
- 输出文件：联络员名单

#### 二、核心管理资源配置流程

**步骤3：任职条件确定**
- 发起部门：公司总部项目管理部
- 操作动作：在确定项目团队人选时，充分考虑项目特点和业主需求，确定项目管理团队核心人员的任职条件
- 输出文件：任职条件清单

**步骤4：人才筛选**
- 发起部门：公司总部项目管理部/分公司/事业部
- 操作动作：从人才库中筛选团队核心人员
- 优先条件：具备同类工程经验且业绩较好的员工优先选聘
- 输出文件：候选人名单

**步骤5：配置决策**
- 发起部门：公司总部/分公司/事业部
- 操作动作：根据任职条件确定最终配置
- 分级管理：重大项目/重点管控项目→总部牵头；一般项目→分公司/事业部负责
- 输出文件：核心管理人员汇总表
- 签字审批：相应的责任领导（履约副总经理/总工程师/总经济师）

**步骤6：汇总表编制**
- 发起部门：项目部
- 操作动作：项目团队开展首次资源配置工作，编制核心管理人员汇总表
- 输出文件：《首次资源配置汇总表》

#### 三、核心技术资源配置流程

**步骤7：技术资源提取**
- 发起部门：项目部/公司科技质量部
- 操作动作：以公司现有科技成果资源平台为依托，提取/借鉴相关技术成果
- 通过对比/优化，确定各类技术方案
- 输出文件：技术方案清单

**步骤8：属地条件调查**
- 发起部门：项目部
- 操作动作：调查项目属地条件（业主对工程产品及施工生产的合理要求/项目属地行政管理要求/国家现行有关标准/地质和水位条件等）
- 输出文件：属地条件调查报告

**步骤9：施工方案清单编制**
- 发起部门：项目部
- 操作动作：编制主要施工方案清单，明确关键要素，提供核心生产资源配置的技术条件
- 输出文件：主要施工方案清单

**步骤10：汇总表编制**
- 发起部门：项目部
- 操作动作：编制核心技术资源汇总表
- 输出文件：《首次资源配置汇总表》（核心技术部分）

#### 四、核心生产资源配置流程

**步骤11：资源库核查**
- 发起部门：项目部/分公司/事业部/公司商务管理部
- 操作动作：
  - 优先选择现有资源平台中的合格分供/分包商
  - 公司商务管理部/项目管理部/财务资金部负责核心生产资源配置工作
  - 搭建生产资源采购平台的维护和统筹管理
- 输出文件：可选资源清单

**步骤12：新资源引入（必要时）**
- 发起部门：项目部/分公司/事业部
- 操作动作：
  - 特殊情况需引进新的分包商时→严格按照管理制度规定的资源引入程序开展
  - 进行考察/审核→纳入资源平台后方可参与分包采购招投标
- 输出文件：新资源准入资料/审批记录

**步骤13：劳务分包统筹**
- 发起部门：公司总部主管部门
- 操作动作：重点统筹劳务分包的使用，避免一家优质分包被多个项目同时使用造成履约能力下降
- 输出文件：统筹分配记录

**步骤14：属地资源调查**
- 发起部门：项目部
- 操作动作：调查项目属地资源情况（行政管理要求/生产资源和分包资源的渠道/价格/同类项目资源配置习惯等）
- 输出文件：属地资源调查报告

**步骤15：配置决策**
- 发起部门：项目团队核心成员
- 操作动作：项目核心生产资源配置应由项目团队核心成员共同决策，并留有决策记录
- 输出文件：决策记录/核心生产资源汇总表

**步骤16：汇总表编制**
- 发起部门：项目部
- 操作动作：编制核心生产资源汇总表
- 输出文件：《首次资源配置汇总表》（核心生产资源部分）

#### 五、检查及评价流程

**步骤17：过程跟踪**
- 发起部门：项目部
- 操作动作：随履约进程随时关注配置资源的使用状态，对主要资源的履约能力和执行总包意图能力进行观测，记录相关信息和数据
- 作为资源配置效果分析和分供/分包考核评价的依据
- 输出文件：跟踪记录

**步骤18：效果分析**
- 发起部门：项目部
- 操作动作：在每个子分部精益建造措施实施完毕后15天内完成实施效果分析，编制分析报告
- 输出文件：效果分析报告

**步骤19：季度检查评价**
- 发起部门：分公司/事业部
- 操作动作：每季度对所辖项目首次资源配置应用效果进行检查评价
- 针对每个项目的资源保障能力和降本增效情况进行汇总，形成考核报告
- 输出文件：季度考核报告

**步骤20：半年检查评价**
- 发起部门：公司总部项目管理部
- 操作动作：每半年开展一次首次资源配置实施情况的检查评价工作
- 输出文件：半年检查报告

**步骤21：年度汇总考核**
- 发起部门：公司总部项目管理部
- 操作动作：每年对当年竣工项目资源配置应用效果进行汇总考核
- 输出文件：年度汇总考核报告

**步骤22：倒查（如有）**
- 发起部门：公司总部项目管理部
- 操作动作：如项目部发生重大投诉或其他影响企业品牌的履约管理问题→适时开展首次资源配置管理倒查
- 输出文件：倒查报告

### 常见问题 / 风险点

1. **核心管理人员配置不合理**：项目经理和班子成员任职条件不清晰，人才库筛选走过场
2. **技术资源配置不匹配**：从科技成果平台提取的技术方案与项目实际情况不符
3. **生产资源配置不及时**：分包/物资/设备资源配置滞后于施工进度需要
4. **属地资源调查不充分**：对项目属地的行政管理要求和资源渠道了解不足
5. **一家优质分包被重复使用**：劳务分包统筹不到位，导致履约能力下降
6. **新引进分包商未合规准入**：未经考察审核直接参与采购招投标
7. **效果分析流于形式**：子分部完成后15天内未完成效果分析报告
8. **跟踪记录缺失**：对配置资源的使用状态缺乏持续跟踪记录
